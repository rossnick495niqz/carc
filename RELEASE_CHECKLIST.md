# Чек-лист Релиза (Release Checklist)

Перед каждой выкладкой новой версии в Cloudflare Pages, пройдите эти простые шаги, чтобы убедиться в стабильности приложения.

## 1. Подготовка (Локально)
Выполните команды в терминале проекта:

1. **Получить свежий код**:
   ```bash
   git pull
   ```

2. **Установить зависимости и проверить Node**:
   ```bash
   node -v  # Должно быть v20+
   npm ci
   ```

3. **Запустить тесты**:
   ```bash
   npm test
   ```
   *Ожидаемый результат*: Все тесты (`PASS`) пройдены.

4. **Собрать проект**:
   ```bash
   npm run build
   ```
   *Ожидаемый результат*: Папка `dist` создана, ошибок нет.

## 2. Проверка Сборки (Dist)
Проверьте наличие ключевых файлов в папке `dist/`:

- [ ] `dist/index.html` (Главная страница)
- [ ] `dist/_redirects` (Файл роутинга для Cloudflare)
- [ ] `dist/manifest.webmanifest` (Настройки PWA)
- [ ] `dist/sw.js` (Service Worker для оффлайна)
- [ ] `dist/data/catalog.json` (База данных авто)

**Быстрая проверка**:
```bash
ls -la dist | grep -E "index.html|_redirects|manifest|sw.js"
ls -la dist/data/catalog.json
```

## 3. PWA Иконки
Убедитесь, что иконки сгенерированы:
- [ ] `dist/pwa-192x192.png`
- [ ] `dist/pwa-512x512.png`
- [ ] `dist/maskable-512x512.png`
- [ ] `dist/apple-touch-icon.png`

## 4. Публикация
Если пункты 1-3 пройдены успешно:

1. **Закоммитьте изменения** (если были):
   ```bash
   git add .
   git commit -m "chore: release v1.0"
   ```

2. **Отправьте в репозиторий**:
   ```bash
   git push origin main
   ```

3. **Следите за Deploy**:
   Перейдите в Cloudflare Dashboard или GitHub Actions и дождитесь завершения сборки.

## 5. Проверка Live
После успешного деплоя, откройте сайт:
1. Проверьте версию (обновите страницу).
2. Убедитесь, что нет ошибки "Offline" (если есть интернет).
3. Проверьте расчет пошлин (базовый сценарий).
